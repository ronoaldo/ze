# Plano de Implementação: Refatoração do `git_diff`

## 1. Diagnóstico do Problema Atual (Hipótese)
O erro `[0, 0]` indica que o parser atual está:
1.  Falhando ao somar as linhas de múltiplos arquivos (está pegando apenas a primeira ou a última linha de forma errada).
2.  Ou, o comando executado não é o `git diff --numstat`, e sim um comando que não fornece os contadores de linhas (como um `git status` simples).
3.  A implementação atual não distingue entre arquivos *staged* e *unstaged* de forma eficaz no parsing.

## 2. Análise da Proposta de Solução

### 2.1. Estratégia de Execução
Em vez de um único comando, utilizaremos três comandos específicos para obter dados granulares e precisos. A lógica será baseada em **agregação de contadores**.

| Informação Desejada | Comando Git | Formato de Saída Esperado (`--numstat`) |
| :--- | :--- | :--- |
| **Unstaged** | `git diff --numstat` | `[adição] [deleção] [caminho]` |
| **Staged** | `git diff --numstat --cached` | `[adição] [deleção] [caminho]` |
| **Untracked** | `git status --untracked-files --short` | `?? [caminho]` (para arquivos não rastreados) |

### 2.2. Modelagem de Dados (Interna)
Criaremos uma estrutura interna para centralizar os dados antes da formatação:

```go
type GitDiffStats struct {
    UnstagedAdd    int
    UnstagedDel    int
    StagedAdd      int
    StagedDel      int
    UntrackedCount int
}
```

### 2.3. Lógica de Parsing (O "Core" da Correção)
A implementação deve seguir este fluxo para cada comando:

1.  **Para `--numstat` (Unstaged e Staged):**
    *   Executar o comando.
    *   Iterar sobre cada linha da saída.
    *   Usar `strings.Fields(line)` para separar as colunas.
    *   Converter as duas primeiras colunas para `int` (tratando o caso de arquivos binários que retornam `-` em vez de números).
    *   Acumular os valores nos campos correspondentes da struct `GitDiffStats`.

2.  **Para `status --short` (Untracked):**
    *   Executar o comando.
    *   Contar quantas linhas começam com `??`.
    *   Atribuir ao `UntrackedCount`.

## 3. Regras de Formatação (UI)
A string final deve ser construída dinamicamente para evitar elementos vazios (ex: `[+0/-0]`).

**Algoritmo de construção da string:**
1.  Iniciar com o prefixo: `git_diff('.')`
2.  **Se** `UnstagedAdd + UnstagedDel > 0`:
    *   Adicionar ` [+<add>/-<del>]`
3.  **Se** `StagedAdd + StagedDel > 0`:
    *   Adicionar ` [+<add>/-<del> staged]`
4.  **Se** `UntrackedCount > 0`:
    *   Adicionar ` <n> new file`
5.  Retornar a string formatada.
