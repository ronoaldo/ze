# Plano de Implementação: Modo Headless (Stdin)

## Objetivo
Permitir que o Zé seja utilizado em ambientes não interativos (scripts shell, pipes, redirecionamentos), detectando automaticamente se o `os.Stdin` é um terminal (TTY).

## Comportamento Esperado (Headless Mode)
1. **Detecção Automática**: Se `os.Stdin` não for um terminal (TTY), o Zé entra em modo "headless".
2. **Sem Banner**: O banner de boas-vindas (neofetch) não deve ser exibido.
3. **Prompt Simplificado**: O prompt deve ser `prompt > ` em vez de `ze > `.
4. **Sem Cores**: Todas as sequências de escape ANSI devem ser desativadas (mesmo que o usuário tenha passado `--no-color`, o modo headless deve forçar isso).
5. **Echo de Input**: Ao ler uma linha do `stdin`, o Zé deve imprimir no `stdout` a linha precedida por `prompt > ` para que o usuário/script saiba o que foi enviado.
6. **Saída Limpa**: O programa deve encerrar graciosamente ao encontrar um `EOF` (End of File).

## Estratégia de Desenvolvimento (TDD)

O desenvolvimento seguirá o ciclo: *Red (Escrever Teste Falhando) -> Green (Implementar) -> Refactor (Refatorar)*.

### Fase 1: Testes de Unidade (Red)
Criar `internal/tui/headless_test.go` para validar a lógica antes da implementação.

**Cenários de Teste:**
1. `TestTUI_HeadlessPrompt`: Injetar um `io.Reader` (via `strings.NewReader`) em um `TUI` configurado como headless e verificar se o prompt exibido é `prompt > `.
2. `TestTUI_HeadlessNoColors`: Verificar que, em modo headless, a `Palette` utilizada é a `NoColorPalette`.
3. `TestTUI_HeadlessEcho`: Verificar se o `input` lido é impresso no `stdout` no formato `prompt > <input>`.
4. `TestTUI_HeadlessEOF`: Verificar se o loop de execução do `TUI.Run` encerra corretamente ao receber `EOF`.

### Fase 2: Implementação (Green)

#### 1. Modificação em `internal/tui/tui.go`
*   **Struct `TUI`**: Adicionar campo `isHeadless bool`.
*   **Função `New`**: 
    *   Implementar detecção de TTY: `isTTY := isatty.IsTerminal(int(os.Stdin.Fd()))` (ou implementação via `syscall`).
    *   Se não for TTY, definir `isHeadless = true` e forçar `palette = NoColorPalette()`.
*   **Método `Run`**:
    *   Ajustar a lógica de impressão do prompt para usar `prompt > ` se `isHeadless` for verdadeiro.
    *   Após o `readLine()`, se `isHeadless` for verdadeiro, imprimir `fmt.Fprintf(t.w, "prompt > %s\n", input)`.

#### 2. Modificação em `cmd/ze/main.go`
*   Ajustar a lógica de exibição do banner. O `printNeofetch` deve ser condicional ao fato de o Zé estar em modo interativo (podemos passar essa informação via `cfg` ou verificar o TTY antes de chamar o `printNeofetch`).

### Fase 3: Refatoração (Refactor)
*   Garantir que a lógica de detecção de TTY esteja isolada.
*   Verificar se o `InputHandler` (criado no plano 02.1) não conflita com o novo comportamento de echo do `TUI`.

### Fase 4: Validação Final (Integration)
*   Executar `go test ./... -v` para garantir que nenhum comportamento existente foi quebrado.
*   **Teste Manual 1 (Interativo):** Rodar `./ze` e verificar banner, cores e prompt `ze > `.
*   **Teste Manual 2 (Pipe):** Rodar `echo "comando" | ./ze` e verificar se o output contém `prompt > comando` e a resposta, sem cores ou banner.
*   **Teste Manual 3 (Redirecionamento):** Rodar `./ze < arquivo_com_comandos.txt` e verificar o comportamento de finalização.
