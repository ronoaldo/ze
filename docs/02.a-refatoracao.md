# Plano de Refatoração: Extração do InputHandler para Testabilidade

## Objetivo
Extrair a lógica de decisão do loop de input (que atualmente reside no `main.go`) para um componente isolado e testável dentro de `internal/tui`. Isso permitirá a implementação da feature `/multiline` utilizando TDD em uma sessão futura.

## 1. Implementação do Componente

### Arquivo: `internal/tui/input_handler.go`

Criar uma struct `InputHandler` para gerenciar o estado do input.

**Estrutura proposta:**
```go
type InputHandler struct {
    isMultiline     bool
    multilineBuffer strings.Builder
    // Dependências injetadas para permitir testes unitários
    commandExecutor func(string) (string, error)
    agentExecutor   func(string) (string, agent.AgentStats, error)
}
```

**Método Principal:**
`func (h *InputHandler) Process(input string) (string, agent.AgentStats, error)`

**Lógica de Estados:**
1.  **Modo Normal (`isMultiline == false`):**
    *   Se `input == "/multiline"`: 
        *   Mudar `isMultiline = true`.
        *   Retornar a mensagem de instrução ao usuário.
    *   Se `input` for um comando válido (via `commandExecutor`):
        *   Retornar o resultado do comando.
    *   Caso contrário:
        *   Chamar o `agentExecutor(input)` e retornar o resultado.

2.  **Modo Multiline (`isMultiline == true`):**
    *   Se a linha começa com `"/send"`:
        *   Extrair o conteúdo acumulado no `multilineBuffer`.
        *   Resetar o buffer e mudar `isMultiline = false`.
        *   Chamar `agentExecutor(conteudo_acumulado)` e retornar o resultado.
    *   Se não:
        *   Acumular a linha atual no `multilineBuffer` (incluindo `\n`).
        *   Retornar string vazia e `AgentStats{}` (para não poluir o TUI enquanto o usuário digita).

## 2. Estratégia de Testes

### Arquivo: `internal/tui/input_handler_test.go`

Implementar testes de unidade utilizando "mocks" (funções anônimas) para `commandExecutor` e `agentExecutor`.

**Cenários de Teste:**
*   `TestInputHandler_NormalCommands`: Verifica se comandos normais são delegados corretamente ao `commandExecutor`.
*   `TestInputHandler_AgentMessages`: Verifica se mensagens de texto comuns são delegadas ao `agentExecutor`.
*   `TestInputHandler_MultilineActivation`: Verifica se o comando `/multiline` altera o estado interno.
*   `TestInputHandler_MultilineAccumulation`: Verifica se, no modo multiline, o texto é acumulado no buffer sem disparar o agente.
*   `TestInputHandler_MultilineCompletion`: Verifica se o comando `/send` dispara o agente com o texto completo e restaura o modo normal.

## 3. Integração

### Arquivo: `cmd/ze/main.go`

1.  Instanciar o `InputHandler` dentro da função `main`.
2.  Passar as funções reais do projeto:
    *   `commandExecutor`: `commands.ExecuteCommand`
    *   `agentExecutor`: O closure que contém `zeAgent.Run`.
3.  Substituir a lógica dentro de `t.Run` por uma chamada direta a `handler.Process(msg)`.
